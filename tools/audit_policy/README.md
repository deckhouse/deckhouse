# audit_policy generator

The generator in `tools/audit_policy` produces:

1. A Go file with basic audit policy targets (namespaces/serviceaccounts).
2. Autogenerated blocks inserted into `KUBERNETES-API-AUDIT*` documentation pages.
3. Fully autogenerated detailed rules pages: `AUDIT-RULES*`.

## Data sources

### 1) Built-in audit policy rules and descriptions
Source: `modules/040-control-plane-manager/hooks/audit_policy.go`

The generator uses `BuiltInAuditPolicyRulesForDocumentation()` which returns:
- built-in `audit.PolicyRule` entries,
- RU/EN rule descriptions.

This is the single source of truth for generated audit-policy documentation content.

### 2) System namespaces and serviceaccounts for the basic policy target lists
Source: repository module scanning (`module.yaml`, `.namespace`, `templates/rbac-for-us.yaml`).

Implemented in `tools/audit_policy/main.go` via:
- `findModules()`
- `processModuleDefinition()`
- `findModuleTemplatesDirs()`
- `processServiceAccounts()`

## Output targets

### Generated Go code
- `modules/040-control-plane-manager/hooks/audit_policy_basic_targets_generated.go`

### Updated autogenerated blocks in existing docs pages
- `docs/site/pages/virtualization-platform/documentation/admin/platform-management/security/KUBERNETES-API-AUDIT.md`
- `docs/site/pages/virtualization-platform/documentation/admin/platform-management/security/KUBERNETES-API-AUDIT_RU.md`
- `docs/documentation/pages/admin/configuration/security/KUBERNETES-API-AUDIT.md`
- `docs/documentation/pages/admin/configuration/security/KUBERNETES-API-AUDIT_RU.md`

### Fully autogenerated detailed pages
- `modules/040-control-plane-manager/docs/AUDIT-RULES.md`
- `modules/040-control-plane-manager/docs/AUDIT-RULES_RU.md`

## Templates (GoTemplate)

Templates are stored in `tools/audit_policy/templates/`:

- Short embedded block templates:
  - `short_block_en.gotmpl`
  - `short_block_ru.gotmpl`

- Detailed full-page templates:
  - `detailed_rules_page_en.gotmpl`
  - `detailed_rules_page_ru.gotmpl`

Note: the generator currently uses the language-specific `*_en.gotmpl` / `*_ru.gotmpl` templates.

## How to run

Recommended command:

```bash
go generate ./tools/audit_policy.go
```

`go:generate` is defined in `tools/audit_policy.go` and runs:

```bash
go run ./tools/audit_policy/main.go \
  -output-go ... \
  -output-doc-en ... \
  -output-doc-ru ... \
  -output-doc-en-2 ... \
  -output-doc-ru-2 ... \
  -output-rules-en ... \
  -output-rules-ru ...
```

## What to edit for different change types

- If you change the rule set/logic/semantics: edit `modules/040-control-plane-manager/hooks/audit_policy.go`.
- If you change docs formatting/text: edit templates in `tools/audit_policy/templates/`.
- If you change output routing or generation flags: edit `tools/audit_policy.go` and/or `tools/audit_policy/main.go`.

After any changes, run:

```bash
go generate ./tools/audit_policy.go
```
