# Вебхуки конфигурации

Вебхуки в поддиректории `deckhouse-config` отвечают за обеспечение наличия и целостности критически важных CRD (Custom Resource Definitions) в кластере. Они гарантируют, что необходимые пользовательские ресурсы могут быть созданы и использованы другими компонентами системы.

## Список вебхуков

- вебхук 1
    - название: ensure_crd.go
    - назначение: Гарантирует, что CRD для `ModuleConfig` существует в кластере, создавая его при необходимости из файла `module-config.yaml`.
    - пример использования: Запускается на старте и проверяет наличие CRD `moduleconfigs.deckhouse.io`, создавая его, если он отсутствует, что является предварительным условием для работы других модулей.
