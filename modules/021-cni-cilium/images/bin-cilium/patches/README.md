## 018-bugtool-remove-sensitive-files.patch

Verbatim cherry-pick of the upstream fix for **CVE-2026-41520** (GHSA-gj49-89wh-h4gj): `cilium-bugtool` was archiving the entire state directory, which on nodes with WireGuard Transparent Encryption enabled included `cilium_wg0.key` — leaking the node's WireGuard private key in any bugtool / `cilium sysdump` archive.

The patch removes `*.key` files from the state directory copy before archiving.

- Advisory: <https://github.com/cilium/cilium/security/advisories/GHSA-gj49-89wh-h4gj>
- Upstream backport into release-1.17 branch: <https://github.com/cilium/cilium/commit/3e26bef96e1ec6ea1b5034fee3f0bc6ddfabcae6>
- Released in cilium v1.17.15 / v1.18.9 / v1.19.3.

**Remove this patch when the base cilium version in `modules/021-cni-cilium/oss.yaml` is bumped to v1.17.15 or newer** — the fix is then already in the upstream sources and this patch will fail to apply.

## 019-fix-CVE-2026-49445.patch

Cherry-pick of the upstream fix for **CVE-2026-49445**: the Envoy admin socket (`admin.sock`) was created world-accessible, allowing sensitive information disclosure and cluster disruption via the local Envoy admin interface. The fix sets mode `0660` on the socket.

Only the Go part (`pkg/envoy/embedded_envoy.go`) is kept. Deckhouse runs the **embedded** Envoy (built into the agent image), whose bootstrap config is generated by `writeBootstrapConfigFile`. The upstream commit also patches `install/kubernetes/cilium/files/cilium-envoy/configmap/bootstrap-config.yaml`, which belongs to the upstream Helm chart for the standalone `cilium-envoy` DaemonSet — that file is not shipped in Deckhouse, so it is intentionally excluded.

- Upstream backport into release-1.17 branch: <https://github.com/cilium/cilium/commit/12fededc90de24c086e62f2f653b07ce7a0338c7>
- Released in cilium v1.17.14 / v1.18.8 / v1.19.2.

**Remove this patch when the base cilium version in `modules/021-cni-cilium/oss.yaml` is bumped to v1.17.14 or newer.**

## 020-fix-CVE-2025-64715.patch

Cherry-pick of the upstream fix for **CVE-2025-64715**: a `toGroups`/`fromGroups` reference in a CNP/CCNP that resolved to an empty set of IP addresses (e.g. a non-existent security group) produced a derived policy with no L3 selector, silently degrading to allow-all at L3 and leading to unrestricted egress traffic. The fix changes the `{To,From}CIDRSet` field tags from `omitempty` to `omitzero` so nil and empty slices marshal to the correct user intent.

This is a verbatim cherry-pick of the upstream commit, including the accompanying `*_test.go` changes.

- Upstream backport into release-1.17 branch: <https://github.com/cilium/cilium/commit/a7613a8b609b3448fb1080ed01d9ba4a966bde02>
- Released in cilium v1.17.10 / v1.18.4 / v1.16.17.

**Remove this patch when the base cilium version in `modules/021-cni-cilium/oss.yaml` is bumped to v1.17.10 or newer.**

## 021-fix-CVE-2026-53935.patch

Cherry-pick of the upstream fix for **CVE-2026-53935**: a `CiliumLocalRedirectPolicy` with an `addressMatcher` could override an existing Service sharing the same frontend address, enabling cross-namespace service traffic hijacking and corrupting the service map on delete. The fix refuses such an override unless the existing entry is itself a `LocalRedirect` service (gated behind the new hidden `enable-lrp-address-matcher-override` option, which stays `false`).

The `redirectpolicy/manager.go` hunk was manually adapted to sit on top of patch `010-fix-cilium-local-redirect-policy.patch`, which already modifies the same `upsertService` function; the `*_test.go` and docs changes from the upstream commit are omitted. This patch must be applied **after** patch 010.

- Upstream backport into release-1.17 branch: <https://github.com/cilium/cilium/commit/70de38b81e>
- Released in cilium v1.17.16 / v1.18.10 / v1.19.4.

**Remove this patch when the base cilium version in `modules/021-cni-cilium/oss.yaml` is bumped to v1.17.16 or newer.**
