# Особенности

## Работа режимов

1. Direct, Unmanaged — могут запускаться только в кластерах, управляемых DKP;
1. Local, Proxy — могут запускаться только в static-кластерах.

## Bootstrap

1. Если в кластере указано `clusterIsBootstrapped: false`, смена параметров registry заблокирована. Так как deckhouse находится в host network, необходимо, чтобы registry также был в host network (для обращения к API registry). Из-за этого при смене режимов может появиться коллизия по портам registry. Поэтому модуль «замораживает» input-параметры и блокирует их смену. Для разблокировки необходимо создать worker-ноду.

## Переключение режимов

1. Заблокированы следующие переключения:
   - Local -> Proxy;
   - Proxy -> Local;
   - Local -> неконфигурируемый Unmanaged.
