Выбранное имя будет соотвествовать имени модуля в Deckhouse. В различных местах оно может быть написано либо с в формате kebab case, либо camel case. В дальшейнем в инструкции нужно использовать то же имя, что вы выбрали.

Откройте `Chart.yaml`, в поле name впишите `hello-world`.

```sh
sed -Ei '' 's/^name:(.*)/name: hello-world/g' Chart.yaml
```

## Подготовьте шаблоны

1. Склонируйте исходный код чарта для hello-world.

   ```sh
   git clone https://github.com/giantswarm/hello-world-app .tmp-chart
   ```

2. Скопируйте шаблоны.

   ```sh
   rm -rf templates/*
   cp -fR .tmp-chart/helm/hello-world/templates/ templates/
   ```

3. Замените в шаблонах путь `.Values` на `.Values.helloWorld`.

   > **NOTE:** Это историческое соглашение в addon-operator для обращения к values модуля. В будущем планируется дать возможность отказаться от этой архитектурной особенности.

   ```sh
   sed -i '' -e 's/.Values/.Values.helloWorld/g' $(find templates/ -type f)
   ```

## Добавьте схему для настроек

Чтобы пользователь мог настраивать модуль, необходимо добавить Open API схему для возможных опций. Это позволит запретить пользователю вводить неверные настройки.

> **NOTE:** Инженеры команды Deckhouse стараются с умом подойти к выбору параметров, которые может настраивать пользователя. Наша философия - продумать верные сценарии за пользователя и дать ему настраивать только то, что действительно нужно.

В чарте hello-world-app уже есть json-схема, поэтому просто сконвертируем ее.

```sh
yq -P .tmp-chart/helm/hello-world/values.schema.json > openapi/config-values.yaml
```

Если в вашем чарте нет схемы, необходимо написать ее самостоятельно. Посмотреть примеры схем можно в репозитории, который вы склонировали на первом шаге.
