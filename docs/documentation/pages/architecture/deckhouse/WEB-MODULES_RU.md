## Модуль console

Модуль [`console`](/modules/console/) — это реализация веб-интерфейса Deckhouse Kubernetes Platform (DKP), которая упрощает управление платформы и делает состояние системы наглядным.

### Архитектура модуля

{% alert level="info" %}
Для упрощения схемы приняты следующие допущения:

* На схеме показано, что контейнеры разных подов взаимодействуют друг с другом напрямую. Фактически они взаимодействуют через соответствующие сервисы Kubernetes (внутренние балансировщики). Названия сервисов не указываются, если они очевидны из контекста. В остальных случаях название сервиса указано над стрелкой.
* Поды могут быть запущены в нескольких репликах, однако на схеме все поды изображены в одной реплике.
{% endalert %}

### Компонтенты модуля

Модуль состоит из следующих компонент:

1. **Frontend** — состоит из одного контейнера **frontend** и реализует веб-интерфейс пользователя и администратора платформы DKP.

1. **Backend** — состоит из одного котнейнера **backend** и реализует API-интерфейс управления ресурсами DKP, получения метрик и логов из других модулей.

1. **Nginx** — проксирует запросы к Grafana API для встраивания дашбордов в основной UI платформы, а так же к **enforcer** для работы с метриками и логами выбранного проекта. 

1. **Dex-authenticator** — обеспечивает проверку и аутентификацию пользователя с использованием единой системы аутентфикации платформы, реализуемой модулем [`user-authn`](/modules/authn). Подробнее с описанием архитектуры модуля `user-authn` можно ознакомиться [в соответствующем разделе документации](../iam/user-authn.html).

### Взаимодействия модуля

Модуль взаимодействует со следующими компонентами:

1. **Kube-apiserver**:
   - организация подключения к ВМ по VNC и TTY;
   — создание, удаление, изменение и отслеживание ресурсов DKP.

1. **Upmeter** — получает информацию о доступности платформы DKP.

1. **Deckhouse-tools** — проксирует запросы для загрузки утилиты [Deckhouse CLI](/cli/d8/).

1. **Aggregation-proxy** — получает информацию по системным метрикам (CPU, RAM, и др.) для быстрой оценки проблем в платформе DKP.

1. **Enforcer** — получает информацию о метриках и логах для выбранного проекта.

С модулем взаимодействуют следующими внешними компонентами:

* **Controller nginx** — проксирует внешние запросы пользователя к веб-интерфейсу модуля.

## Модуль documentation

Модуль [`documentation`](/modules/documentation/) — это реализация веб-интерфейса с документацией, соответствующей запущенной версии Deckhouse Kubernetes Platform.

Названия сервисов не указываются, если они очевидны из контекста. В остальных случаях название сервиса указано над стрелкой.
* Поды могут быть запущены в нескольких репликах, однако на схеме все поды изображены в одной реплике.

### Компонтенты модуля

Модуль состоит из следующих компонентов: 

1. **Documentation** — компонент, обеспечивающий веб-интерфейс документации.

    Состоит из следующих контейнеров:

    * **web** — основной контейнер;

    * **kube-rbac-proxy** — сайдкар-контейнер с авторизующим прокси на основе Kubernetes RBAC для организации защищенного доступа к основному контейнеру. Является [Open Source-проектом](https://github.com/brancz/kube-rbac-proxy);

    * **builder** — сайдкар-контейнер, обеспечивающий расширение документации при установке новых модулей в платформу. **Builder** создаёт и регулярно обновляет объект Lease для объявления обработчика генерации документации.

1. **Dex-authenticator** — обеспечивает проверку и аутентификацию пользователя с использованием единой системы аутентфикации платформы, реализуемой модулем [`user-authn`](/modules/authn). Подробнее с описанием архитектуры модуля `user-authn` можно ознакомиться [в соответствующем разделе документации](../iam/user-authn.html).

### Взаимодействия модуля

Модуль взаимодействует со следующим компонентом:

* **Kube-apiserver**:
   - для обновление ресурса Lease;
   - для авторизации запросов к веб-интерфейсу документации.

С модулем взаимодействуют следующие внешние компоненты:

1. **Deckhouse** — отправляет запрос на обновление документации при изменении состава модулей.

1. **Prometheus** — собирает метрики модуля.

1. **Controller nginx** — проксирует внешние запросы пользователя к веб-интерфейсу модуля.
