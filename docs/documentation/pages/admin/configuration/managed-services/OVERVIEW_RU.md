Managed-сервисы в Deckhouse Kubernetes Platform позволяют запускать в кластере готовые к использованию прикладные сервисы, например базы данных, очереди сообщений и сервисы обработки больших данных, без необходимости самостоятельно управлять их жизненным циклом. DKP берёт на себя технические задачи администрирования, масштабирования, резервного копирования и обновления, а администратор управляет сервисами на уровне классов, ограничений и архитектурных решений.

## Как это работает

Каждый managed-сервис добавляет в кластер собственные ресурсы Kubernetes и контроллеры. Пользователь описывает желаемое состояние сервиса в неймспейсе. Контроллер создаёт и обновляет связанные объекты: рабочие экземпляры, сервисы, секреты, снимки и другие ресурсы, которые поддерживает конкретный managed-сервис. Администратор управляет cluster-wide-ресурсами, которые задают допустимые конфигурации и значения по умолчанию для пользовательских ресурсов.

Используйте этот раздел, чтобы:

- подготовить кластер к работе с managed-сервисами;
- настроить общие параметры и ограничения managed-сервиса;
- определить доступные классы и конфигурации сервисов;
- проверить готовность инфраструктуры к эксплуатации managed-сервисов.

Администрирование сервиса PostgreSQL описано в разделе [Managed PostgreSQL](postgres.html).

Инструкции по созданию сервисов, настройке экземпляров и подключению к ним приведены в разделе [«Использование»](../../../user/managed-services/).
