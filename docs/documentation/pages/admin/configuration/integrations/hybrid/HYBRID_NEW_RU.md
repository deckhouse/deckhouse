Гибридный кластер — это кластер DKP, узлы которого расположены в разных сетях или на разных типах инфраструктуры. Например, он может объединять физические серверы в дата-центре, облачные виртуальные машины и локальные виртуальные машины в единую Kubernetes-среду. 

Такой подход позволяет использовать преимущества разных площадок одновременно: размещать критичные сервисы в собственной инфраструктуре, масштабировать нагрузку в облаке и подключать удалённые площадки без развёртывания отдельных кластеров.

В данный момент поддерживается интеграция с облаками на базе <блаблабла>

## Когда использовать гибридную архитектуру

Гибридная архитектура особенно полезна в случаях, когда необходимо:

- постепенно мигрировать сервисы из локальной инфраструктуры в облако;
- объединить ресурсы нескольких дата-центров или филиалов;
- обеспечить отказоустойчивость за счёт распределения узлов между площадками;
- соблюдать требования к хранению данных и размещать часть сервисов в определённой среде;
- использовать уже существующую инфраструктуру без полного переезда в новое окружение.

При этом построение гибридного кластера требует учитывать сетевую связанность между узлами, задержки, маршрутизацию, безопасность межузлового трафика и особенности размещения рабочих нагрузок.

DKP позволяет создавать такие кластеры и управлять ими как единой платформой, сохраняя стандартные механизмы Kubernetes для развёртывания приложений, масштабирования и эксплуатации.
