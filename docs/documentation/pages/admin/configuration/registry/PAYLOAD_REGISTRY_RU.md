В DKP можно развернуть пользовательское хранилище образов контейнеров, соответствующее стандарту OCI. Для этого используется модуль [`payload-registry`](/modules/payload-registry/).

## Когда использовать этот сценарий

Этот сценарий подходит, если вам нужно:
- хранить образы пользовательских приложений внутри своей инфраструктуры;
- использовать отдельное хранилище образов контейнеров для команд, сервисов или CI/CD;
- сократить зависимость от внешнего хранилища образов контейнеров в процессах доставки приложений [5].

## Что важно не перепутать

`payload-registry` и хранилище образов компонентов DKP — это разные сценарии:

- `payload-registry` используют для образов пользовательских приложений;
- хранилище образов компонентов DKP используют для образов самой платформы.

Если вам нужно настроить хранилище образов для компонентов DKP, используйте другие разделы:
- [«Настройки в кластере, управляемом DKP»](./dkp-component/managing-interaction.html);
- [«Настройки в Managed Kubernetes-кластере»](./dkp-component/third-party.html) [7].

## Что дает модуль `payload-registry`

Модуль `payload-registry` реализует пользовательское хранилище образов контейнеров внутри DKP. Оно предназначено для типовых сценариев работы с образами приложений:
- хранения образов в инфраструктуре компании;
- использования образов в кластере;
- работы с репозиториями и тегами;
- управления доступом к хранилищу.
