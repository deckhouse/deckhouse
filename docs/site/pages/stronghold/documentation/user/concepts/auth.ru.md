Аутентификация в Stronghold — это процесс, при котором информация,
предоставленная пользователем или машиной, проверяется в внутренней или
внешней системе. Stronghold поддерживает несколько [методов аутентификации](/docs/auth),
включая GitHub, LDAP, AppRole и другие. Каждый метод аутентификации
имеет свои специфические случаи применения.
Перед тем как клиент сможет взаимодействовать с Stronghold, он должен
_авторизоваться_ через метод аутентификации. При аутентификации
генерируется токен. Этот токен концептуально похож на идентификатор
сессии на веб-сайте. Токен может иметь прикреплённые политики, которые
отображаются во время аутентификации. Этот процесс подробно описан в
документации [концепции политик](/docs/concepts/policies).

## Методы аутентификации
Stronghold поддерживает ряд методов аутентификации. Некоторые бекенды
предназначены для пользователей, а другие — для машин. Большинство бекендов
аутентификации необходимо включить перед использованием. Чтобы
включить метод аутентификации:

```shell-session
$ d8 stronghold write sys/auth/my-auth type=userpass
```

Это включает метод аутентификации "userpass" на пути "my-auth". Эта
аутентификация будет доступна на пути "my-auth". Зачастую вы увидите
аутентификации на том же пути, что и их название, но это не обязательно.
Чтобы узнать больше об этой аутентификации, используйте встроенную
команду `path-help`:

```shell-session
$ d8 stronghold path-help auth/my-auth
# ...
```

Stronghold поддерживает несколько методов аутентификации одновременно, и
вы даже можете разместить один и тот же тип метода аутентификации на различных
путях. Для получения доступа к Stronghold требуется только одна
аутентификация, и в настоящее время невозможно заставить пользователя
пройти через несколько методов аутентификации для доступа, хотя некоторые
бекенды поддерживают MFA.

## Токены

Есть [целая страница, посвященная токенам](/docs/concepts/tokens),
но важно понимать, что аутентификация работает, проверяя вашу
сущность и затем создавая токен для связи с этой сущностью.
Например, даже если вы аутентифицируетесь с помощью, скажем, GitHub,
Stronghold генерирует уникальный токен доступа для использования в
будущих запросах. CLI автоматически прикрепляет этот токен к запросам,
но если вы используете API, вам придётся делать это вручную.

Этот токен, предоставленный для аутентификации с любым бекендом, также
можно использовать для полного набора команд токенов, таких как создание
новых субтокенов, отзыв токенов и продление токенов. Это все покрыто на
[странице концепции токенов](/docs/concepts/tokens).

## Аутентификация

### Через CLI

Для аутентификации с помощью CLI используется `d8 stronghold login`.
Поддерживается многие методы аутентификации. Например, аутентификация
с помощью OIDC:

```shell-session
$ d8 stronghold login -method=oidc
...
```

После аутентификации вы будете залогинены. Команда CLI также
выведет ваш токен. Этот токен используется для отзыва и продления.
Для пользователя, осуществляющего вход, основной случай использования
токена — это продление, описанное ниже в разделе "Срок действия авторизации".
Чтобы определить, какие переменные нужны для метода аутентификации,
используйте флаг `-method` без дополнительных аргументов, и будет показана
помощь.
Если вы используете метод, который не поддерживается через CLI, то необходимо
использовать API.

### Через API

Аутентификация через API обычно используется для аутентификации
машин. Каждый метод аутентификации реализует свою конечную точку логина.
Используйте механизм `d8 stronghold path-help`, чтобы найти правильный эндпоинт.

## Срок действия авторизации

Так же как и секреты, сущности имеют сроки действия аренды ,
связанные с ними. Это означает, что вы должны пройти повторную аутентификацию
после указанного срока аренды, чтобы продолжить доступ к Stronghold.

Чтобы установить срок аренды, связанный с сущностью, обратитесь к
справке по конкретному используемому методу аутентификации. В каждом бекенде
реализована своя система продления срока аренды.

И так же как и секреты, сущности могут быть продлены без необходимости
полностью повторно аутентифицироваться. Просто используйте `d8 stronghold token renew <token> `
с токеном, связанным с вашей сущностью, чтобы продлить его.
